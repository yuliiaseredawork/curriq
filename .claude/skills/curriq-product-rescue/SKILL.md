# Curriq Product Rescue

Curriq is an AI adaptive learning platform. The goal is not to expose features. The goal is to help a learner know exactly what to do next and feel real learning progress.

## Core Principle

Do not optimize individual features first.

Optimize the golden path:

1. User adds learning content.
2. Curriq generates a useful course.
3. User sees a clear next action.
4. User completes a learning session.
5. User receives specific feedback.
6. User knows what to do next.

## When diagnosing Curriq, evaluate five layers:

### 1. Product clarity
- Is it obvious who this is for?
- Is the main promise clear?
- Does the product feel like a learning coach or a content generator?

### 2. UX flow
- Does the user know what to do next?
- Is there one primary action?
- Are internal mechanics hidden?
- Are there too many competing concepts?

### 3. AI-generated content quality
- Does the generated course feel specific to the source?
- Are chapters meaningful?
- Are quizzes application-oriented?
- Are distractors plausible?
- Is feedback useful?
- Are flashcards non-trivial?

### 4. UI quality
- Does the interface feel trustworthy?
- Is the hierarchy clear?
- Does it look like a real product?
- Are empty/loading/error states polished?
- Is the layout text-heavy?

### 5. Retention loop
- Does the product make the user want to return tomorrow?
- Does the user understand progress?
- Does the product create confidence?

## Rescue Process

When asked to improve Curriq:

1. Identify the weakest layer.
2. Pick one user persona and one use case.
3. Define the golden path.
4. Rewrite the information architecture.
5. Propose one high-quality dashboard.
6. Propose one high-quality learning session.
7. Define quality standards for generated data.
8. Give implementation tasks in priority order.
9. Avoid adding new features unless the core loop is already strong.

## Output Format

Return:

1. Diagnosis
2. What to stop doing
3. What to fix first
4. Golden path
5. UX changes
6. UI changes
7. AI-content quality changes
8. Concrete Claude prompts to use
9. Implementation checklist
10. Definition of done
